🔢 Data Maintenance
Every family already gets a permanent Family ID (e.g. NSN-0001/1) when created. This one-time action assigns one to existing families that were saved before that — safe to run more than once, and only ever fills in families that don't have one yet.
Scans all existing records and marks any that have family members with missing age or gender as Needs Correction, so workers can fill them in. Safe to run more than once — only records with missing data are flagged.
Some older records may have family members with relation "Head" instead of "Self". This one-time action converts them all to "Self" so the head of family is consistently identified. Safe to run more than once — only members with "Head" are touched.
Some older records were saved before properties were broken into units and have none recorded at all. This one-time action marks those as Needs Correction so workers add at least one unit. Safe to run more than once — only records with zero units are flagged.
⚠️ Danger Zone
Reset Everything permanently deletes all records and all worker profiles. Your admin login stays. This cannot be undone.
